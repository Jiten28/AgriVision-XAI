# AgriVision-XAI — Architecture

## 1. System Architecture Overview

```
Input Image (leaf photo)
        │
        ▼
┌─────────────────────────────────────────────┐
│              PREPROCESSING                  │
│  Resize → 128×128 (CPU) / 224×224 (GPU)    │
│  Augmentation (train only)                  │
│  Cast to float32 [0, 255]                   │
└─────────────────────┬───────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────┐
│         EfficientNetB0 BACKBONE             │
│  Pretrained ImageNet weights                │
│  Phase 1: frozen  │  Phase 2: top-20 open  │
│  Output: (batch, 4, 4, 1280) feature maps  │
└─────────────────────┬───────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────┐
│              CBAM BLOCK                     │
│  Channel Attention → Spatial Attention      │
│  Focuses on disease-relevant regions        │
│  Output: (batch, 4, 4, 1280) — same shape  │
└─────────────────────┬───────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────┐
│          CLASSIFICATION HEAD                │
│  GlobalAveragePool → Dense(256) →           │
│  BatchNorm → ReLU → Dropout(0.4) →         │
│  Dense(38, softmax)                         │
└──────┬────────────────────────┬─────────────┘
       │                        │
       ▼                        ▼
  Disease class            Confidence
  (1 of 38)                  score
       │
       ├──────────────────────────┐
       ▼                          ▼
┌─────────────┐          ┌──────────────────┐
│  XAI MODULE │          │ SEVERITY MODULE  │
│  Grad-CAM   │          │ Grad-CAM mask +  │
│  Grad-CAM++ │          │ HSV segmentation │
│  LIME       │          │ → % infected area│
│  → heatmap  │          │ → mild/mod/severe│
└─────────────┘          └──────────────────┘
```

## 3. Technical Stack

| ----------------- | -------------------------------------------------- |
| Language          | Python 3.13 (CPU) / Python 3.11 in WSL2 (GPU)      |
| Deep Learning     | TensorFlow 2.20 / Keras 3                          |
| Backbone          | EfficientNetB0 (ImageNet pretrained)               |
| Attention         | CBAM (custom Keras Layer, serializable)            |
| Explainability    | Grad-CAM, Grad-CAM++, LIME (custom implementation) |
| Data pipeline     | tf.data with stratified sklearn split              |
| Image processing  | OpenCV, scikit-image, Pillow                       |
| Visualisation     | Matplotlib, Seaborn                                |
| Config management | python-dotenv (.env files)                         |
| Export            | TensorFlow Lite (FP32 + INT8), ONNX (optional)     |
| Frontend          | Streamlit (Phase 3)                                |
| Version control   | Git + GitHub (Jiten28/AgriVision-XAI)              |

## 4. Training Strategy

### Phase 1 — Head Training

- Backbone: **frozen** (ImageNet weights preserved)
- Trains: CBAM block + classification head only
- LR: `1e-3` (safe — only random-init layers receive gradients)
- Epochs: 10 (full) / 2 (CPU smoke test)

### Phase 2 — Fine-tuning

- Backbone: top **20 layers unfrozen**
- LR: `1e-5` (very low — prevents catastrophic forgetting)
- Epochs: up to 30 (with early stopping, patience=7)
- Callbacks: ModelCheckpoint, ReduceLROnPlateau, EarlyStopping, TensorBoard
